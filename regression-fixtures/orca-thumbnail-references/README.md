# Orca Thumbnail Visual References

This directory is for desktop Orca thumbnail PNG references, not MobileSlicer
outputs and not Orca CLI metadata-only outputs.

Expected case layout:

```text
matrix.json
<case>/
  manifest.json
  README.md
  plate.png
  no_light.png
  top.png
  pick.png
  <source desktop Orca .gcode.3mf>
```

`matrix.json` is the source of truth for every checked-in reference case. Both
the matrix regeneration command and the Android visual comparison command read
from it, so adding a case requires adding one entry there and committing the
matching reference directory.

Optional matrix fields:

- `bed`: per-printer bed dimensions used by the Android comparison runner.
- `profiles`: desktop Orca profile paths and display names used by reference
  regeneration.
- `source_layout`: Android comparison source layout. The default is `single`.
  `two_filament_objects` duplicates the matrix model as two independently
  colored thumbnail sources for device-side multi-filament renderer coverage.
- `source_colors`: optional hex colors for `source_layout` entries, for example
  `["#F2754E", "#4EA3F2"]`.

Preferred capture command:

```sh
scripts/generate_orca_thumbnail_reference_fixture.sh
```

Named cases are generated by setting the model and case name:

```sh
ORCA_SLICER_BIN=/path/to/orca-slicer \
MOBILE_SLICER_ORCA_REFERENCE_CASE=tall-box \
MOBILE_SLICER_ORCA_REFERENCE_MODEL=regression-fixtures/slicing/stage2_inner_wall_acceleration_tall_box_fixture.stl \
scripts/generate_orca_thumbnail_reference_fixture.sh
```

Or regenerate every checked-in desktop reference case:

```sh
ORCA_SLICER_BIN=/path/to/orca-slicer \
scripts/generate_orca_thumbnail_reference_matrix.sh
```

Validate the checked-in references without a device:

```sh
scripts/verify_android.sh orca-thumbnail-reference-fixtures
```

That local gate checks that every matrix model exists, every reference manifest
is complete, every source `.gcode.3mf` contains Orca package thumbnail entries,
and every extracted role PNG is a nonblank 512x512 image.

The command writes diagnostics under
`artifacts/orca-thumbnail-reference-capture/<timestamp>/` and fails if the local
Orca run produces metadata-only output without thumbnail PNGs.

Multi-filament note: the Android comparison runner now supports a
`two_filament_objects` source layout so device automation can catch renderer
regressions where object colors or source ordering collapse. That is not yet
listed as a desktop Orca visual-reference case because current CLI probes with
`--load-filaments`, `--load-filament-ids`, and `--load-assemble-list` record
two object/extruder assignments in `Metadata/model_settings.config` but still
write one active filament in `Metadata/plate_1.json` and `slice_info.config`
for the simple cube fixture. Do not claim desktop multi-filament thumbnail
pixel parity until a desktop Orca package proves two active filaments and role
PNGs from the same source package.

Manual extraction from a GUI-capable or patched desktop Orca `.gcode.3mf`
package uses:

```sh
python3 scripts/extract_orca_thumbnail_references.py \
  --source-3mf /path/to/desktop-orca-simple-cube.gcode.3mf \
  --output-dir regression-fixtures/orca-thumbnail-references/simple-cube \
  --orca-build "OrcaSlicer <commit-or-version>" \
  --model regression-fixtures/slicing/mobileslicer_test_cube.stl \
  --printer-profile "Qidi Q2 0.4 nozzle" \
  --filament-profile "Generic PLA @Qidi Q2 0.4 nozzle" \
  --process-profile "0.20mm Standard @Qidi Q2" \
  --allow-overwrite
```

Then make the device visual gate strict:

```sh
MOBILE_SLICER_EGL_COMPARE_MODEL=regression-fixtures/slicing/mobileslicer_test_cube.stl \
MOBILE_SLICER_ORCA_THUMBNAIL_REFERENCE_DIR=regression-fixtures/orca-thumbnail-references/simple-cube \
MOBILE_SLICER_REQUIRE_ORCA_THUMBNAIL_REFERENCE=1 \
MOBILE_SLICER_ALLOW_DEVICE_AUTOMATION=1 \
scripts/verify_android.sh egl-thumbnail-compare <serial>
```

Or run every checked-in case:

```sh
MOBILE_SLICER_ALLOW_DEVICE_AUTOMATION=1 \
scripts/run_orca_thumbnail_reference_matrix.sh [serial]
```

The matrix defaults to 512x512 output to match the desktop Orca reference PNGs.
This matters for tiny full-bed roles: at 128px a long 1.3 mm strip can alias
away even though Orca's 512px package thumbnail is valid.

The matrix is the release-quality Orca thumbnail gate. It requires desktop Orca
references and uses tighter desktop-Orca visual thresholds than the generic
single-fixture `egl-thumbnail-compare` command:

```text
MOBILE_SLICER_ORCA_VISUAL_MAX_LUMA_DELTA=70.0
MOBILE_SLICER_ORCA_VISUAL_MAX_ALPHA_COVERAGE_DELTA=0.08
MOBILE_SLICER_ORCA_VISUAL_MAX_BBOX_DELTA_PX=64
MOBILE_SLICER_ORCA_VISUAL_MIN_COVERAGE_RATIO=0.90
MOBILE_SLICER_ORCA_VISUAL_MAX_COVERAGE_RATIO=1.15
```

The single-fixture command remains useful for debugging a new case, but it is
not enough to claim Orca visual parity by itself. Release automation must run
the full matrix so a renderer change cannot pass by matching only the simple
cube.

Each case rendered `plate`, `no_light`, `top`, and `pick` on Android EGL with
four distinct role hashes and passed the desktop Orca visual diff gate.

The 64 px bbox threshold is deliberate. Coverage ratio, alpha coverage, luma,
nonblank role output, and distinct role hashes remain strict; the wider bbox
tolerance allows desktop Orca's broad angled multi-object framing to differ
slightly from the Android EGL fit without accepting blank, wrong-bed, or badly
cropped thumbnails.

The matrix command auto-selects the only connected ADB device. If no device is
connected, reconnect wireless debugging and rerun it with the current serial.

Use the matching `MOBILE_SLICER_EGL_COMPARE_MODEL` and
`MOBILE_SLICER_ORCA_THUMBNAIL_REFERENCE_DIR` for each case. The comparison is
only meaningful when both point at the same source model.

If local capture is needed on a headless or GL-hostile Linux session, install a
working virtual X server and rerun:

```sh
MOBILE_SLICER_USE_XVFB=1 \
LIBGL_ALWAYS_SOFTWARE=1 \
scripts/generate_orca_thumbnail_reference_fixture.sh
```

If `xvfb-run` is not installed, use a GUI-capable desktop Orca export and the
manual extraction command above. Do not lower the gate to accept missing PNGs.
