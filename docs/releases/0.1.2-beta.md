# MobileSlicer 0.1.2 Beta

MobileSlicer 0.1.2 Beta is an Android beta update for multi-nozzle slicing and
release packaging. It keeps Orca project material assignments intact, restores
Snapmaker U1 multi-nozzle color output, and publishes a fresh APK with a new
version code so testers can install it as a normal update.

This is the recommended beta build for Android testers.

## What's New

- Preserved Orca 3MF project color, material, and extruder assignments through
  slicing instead of flattening imported projects into one single-material STL.
- Restored Snapmaker U1 multi-nozzle output with the expected `filament_map`,
  toolchanges, prime tower geometry, and thumbnails.
- Added an Orca reference fixture for Snapmaker U1 multi-nozzle output and a
  comparator that checks MobileSlicer G-code against the reference.
- Kept STL imports honest: STL files do not carry hidden multicolor data, so
  color assignment must come from app-side painting or object slot assignment.
- Kept Snapmaker U1 `purge_in_prime_tower` behavior aligned with Orca's profile:
  prime tower is enabled by the process profile, while purge-into-tower remains
  disabled by the printer profile.

## Install

Download `MobileSlicer-0.1.2-beta-v5.apk` from the assets below, then open it on
your Android device. Android may ask you to allow installs from your browser or
file manager before continuing.

If you already have an older MobileSlicer beta installed, this build should
install as an update.

## Beta Notes

- Snapmaker U1 multi-nozzle slicing now has an Orca reference fixture and G-code
  audit coverage, but users should still review generated G-code before running
  a long print.
- Bambu LAN printing features remain beta. Connection setup validation is
  stricter, but printer behavior can still vary by model, firmware, and local
  network configuration.
- Thingiverse import work is included in the codebase, but account sign-in and
  direct import may depend on server-side availability during the beta.
- Please report install issues, crashes, printer connection problems, model
  import problems, and incorrect multicolor output through the project feedback
  channels.
