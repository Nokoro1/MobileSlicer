# MobileSlicer 0.1.4 Beta

MobileSlicer 0.1.4 Beta is a release-quality follow-up for Android testers. It
improves multi-nozzle safety checks, makes the public beta download page easier
to verify, and adds stronger release evidence before beta builds are published.

This is the recommended beta build for Android testers.

## Highlights

- Improves release checks for Snapmaker U1 multi-nozzle output.
- Verifies that generated tool changes line up with the expected filament map
  before accepting a multi-slot slice.
- Makes release comparison failures easier to read during beta validation.
- Updates the beta download page with a mobile-friendly checksum block.
- Adds a committed release manifest for later audit.
- Wires GitHub release asset verification into the Android release gate.
- Adds release gates for Bambu LAN printer smoke evidence and Thingiverse
  production OAuth route checks.

## Install

Download `MobileSlicer-0.1.4-beta-v7.apk` from the assets below, then open it on
your Android device. Android may ask you to allow installs from your browser or
file manager before continuing.

If you already have an older MobileSlicer beta installed, this build should
install as an update.

## Verified

- Android unit tests, release lint, release build, APK signing, and 16 KB
  alignment.
- Physical-device install and launch with the signed release APK.
- Snapmaker U1 multi-nozzle G-code validation against the Orca reference fixture,
  including filament mapping, nozzle metadata, tool changes, prime tower
  geometry, and thumbnails.
- GitHub release asset digest, downloaded APK checksum, and Android version
  metadata.

## Beta Notes

- Review generated G-code before running long or unattended prints.
- Bambu LAN printing remains beta. Public release claims should be backed by a
  current real-printer smoke pass for connect, upload-only, upload-and-start,
  wrong access code, and offline-printer behavior.
- Thingiverse account sign-in and direct import remain beta until the production
  OAuth start, callback, and redeem routes pass a live route smoke.
- Report install issues, crashes, printer connection problems, model import
  problems, and incorrect multicolor output through the project feedback
  channels.
